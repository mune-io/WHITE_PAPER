üöñ 2. Decentralized Taxi Application

Status: In Progress ‚öôÔ∏è

Core Concept

Instead of a single central server, we're building a system where computations (match-making, route calculation, driver filtering) are executed on distributed nodes:

  • Advertising tablets (installed in vehicles)
  • Stationary mini-servers
  • Driver and passenger smartphones

How It Works

  1. Passenger places order in the app
  2. Coordinator (lightweight server) distributes task to P2P network
  3. Nearest nodes compute optimal routes and available drivers
  4. Results sent directly to passenger; coordinator logs only key events
